## ⚙️ 首次启动配置（只需一次）

### 步骤1: 修改API地址

**文件**: `admin/.env.development`

**修改第6行**:
```env
# 从
VITE_API_URL = http://localhost:3000/api

# 改为
VITE_API_URL = http://localhost:3001/api
```

**快速修改命令**:
```powershell
(Get-Content admin\.env.development) -replace 'localhost:3000', 'localhost:3001' | Set-Content admin\.env.development
```

---

### 步骤2: 确保数据库有数据

```bash
# 如果数据库是空的，运行这个
cd backend
node migrations/seed-test-data.js
```

## ⚠️ 常见问题

### 问题1: 端口被占用

**症状**:
```
Error: listen EADDRINUSE: address already in use :::3001
```

**解决**:
```bash
# 查找占用端口的进程
netstat -ano | findstr :3001

# 结束进程（替换PID为实际进程ID）
taskkill /PID <PID> /F
```

---

### 问题2: 数据库连接失败

**症状**:
```
Error: connect ECONNREFUSED 127.0.0.1:3306
```

**解决**:
```bash
# 启动MySQL服务
net start MySQL

# 或检查MySQL是否运行
```

---

### 问题3: 前端页面空白

**症状**: 页面加载但没有内容

**解决**:
1. 打开浏览器开发工具（F12）
2. 查看Console错误
3. 查看Network请求
4. 确认API地址配置正确

---

### 问题4: API请求404

**症状**: 
```
GET http://localhost:3000/api/products 404
```

**原因**: API地址配置错误

**解决**: 
1. 检查 `admin/.env.development`
2. 确认是 `localhost:3001` 不是 `localhost:3000`
3. 重启前端服务
