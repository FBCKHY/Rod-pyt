# Art Design Pro 项目更新

## 📋 概述

如果你在基于本项目进行开发，并希望后续持续同步开源仓库的更新，建议按以下方式配置和操作。通过合理的 Git 仓库管理策略，既能保持自己的开发进度，又能及时获取开源项目的最新功能和修复。

## 🚀 代码如何保持最新

### 📥 步骤一：克隆代码源码

首先从官方仓库克隆最新的源码：

```bash
git clone https://github.com/Daymychen/art-design-pro
```

克隆完成后，进入项目目录：

```bash
cd art-design-pro
```

### 🔗 步骤二：添加自己的 Git 仓库作为远程源

将你自己的 Git 仓库添加为远程源，便于后续推送自己的修改：

```bash
# 添加远程源（up 为自定义名称，可自行更改）
git remote add up <your-git-url>
```

**示例：**
```bash
git remote add up https://github.com/yourname/your-art-design-pro.git
```

### 📤 步骤三：推送代码到你自己的仓库

将初始代码推送到你的远程仓库：

```bash
# 将代码推送到你自己的 Git 仓库
# 注意：main 为分支名，请根据你的实际分支进行替换
git push up main
```

### 🔄 步骤四：同步你自己仓库的最新代码（可选）

如果你在多个地方进行开发，可以从自己的远程仓库拉取更新：

```bash
# 从你自己的远程仓库拉取更新
git pull up main
```

### ⬇️ 步骤五：同步开源项目的最新代码（推荐定期执行）

定期从开源仓库同步最新的代码和功能：

```bash
# 从开源仓库拉取最新代码（main 为默认主分支，请按需修改）
git pull origin main
```

## 🔍 查看远程仓库配置

检查当前配置的远程仓库：

```bash
# 查看所有远程仓库
git remote -v
```

预期输出示例：
```
origin  https://github.com/Daymychen/art-design-pro.git (fetch)
origin  https://github.com/Daymychen/art-design-pro.git (push)
up      https://github.com/yourname/your-art-design-pro.git (fetch)
up      https://github.com/yourname/your-art-design-pro.git (push)
```

## 📋 完整的更新工作流

### 日常开发流程

1. **开始开发前**
   ```bash
   # 1. 同步开源项目最新代码
   git pull origin main
   
   # 2. 解决可能的冲突（如果有）
   # 手动解决冲突文件，然后：
   git add .
   git commit -m "解决合并冲突"
   
   # 3. 推送到自己的仓库
   git push up main
   ```

2. **开发过程中**
   ```bash
   # 正常的开发提交流程
   git add .
   pnpm commit  # 使用规范化提交
   git push up main
   ```

3. **定期同步（建议每周）**
   ```bash
   # 同步开源项目的最新更新
   git pull origin main
   git push up main
   ```

### 冲突处理策略

#### 预防冲突
- **定期同步**：建议每周至少同步一次开源项目
- **分支开发**：使用功能分支进行开发，避免直接在 main 分支修改
- **关注更新日志**：查看开源项目的更新内容，了解可能的冲突点

#### 解决冲突
当执行 `git pull origin main` 遇到冲突时：

1. **查看冲突文件**
   ```bash
   git status
   ```

2. **手动解决冲突**
   - 打开冲突文件
   - 选择保留的代码（删除冲突标记）
   - 测试确保功能正常

3. **提交解决结果**
   ```bash
   git add .
   git commit -m "解决合并冲突"
   git push up main
   ```

## 🔧 高级配置

### 使用功能分支进行开发

```bash
# 创建功能分支
git checkout -b feature/new-functionality

# 开发完成后合并到主分支
git checkout main
git merge feature/new-functionality

# 删除功能分支
git branch -d feature/new-functionality
```

### 设置上游仓库（推荐）

```bash
# 设置上游仓库，简化后续操作
git branch --set-upstream-to=origin/main main
```

### 配置 Git 别名

在 `~/.gitconfig` 中添加以下别名，简化操作：

```ini
[alias]
    sync-upstream = pull origin main
    push-mine = push up main
    st = status
    co = checkout
    br = branch
```

使用别名：
```bash
git sync-upstream  # 相当于 git pull origin main
git push-mine      # 相当于 git push up main
```

## ⚠️ 重要注意事项

### 🚨 同步前的准备工作

> **注意：同步更新前建议先 git pull 拉取代码并处理冲突，确保本地变更不会被覆盖。合并后再推送到你的远程仓库。**

### 备份策略

1. **重要修改前备份**
   ```bash
   # 创建备份分支
   git checkout -b backup-$(date +%Y%m%d)
   git checkout main
   ```

2. **定期推送到自己的仓库**
   ```bash
   git push up main
   ```

### 常见问题解决

#### 问题1：推送被拒绝
```bash
# 错误信息：Updates were rejected because the remote contains work
# 解决方案：先拉取再推送
git pull up main
git push up main
```

#### 问题2：合并冲突
```bash
# 如果冲突过多，可以重置到冲突前状态
git merge --abort
# 或者使用强制策略（谨慎使用）
git pull origin main --strategy=recursive -X theirs
```

#### 问题3：忘记添加远程仓库
```bash
# 添加远程仓库
git remote add up <your-git-url>
# 检查是否添加成功
git remote -v
```
